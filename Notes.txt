To keep the repository clean and consistent with its modular design, the **HZD optimization utilities should live inside `Trajectory_Optimization/`**, because that folder already contains:

- `optimize_gait.m`
- `direct_collocation.m`
- `gait_constraints.m`
- `rabbit_hzd_trajectory_optimization.m`

Your new files are **trajectory optimization utilities**, not controllers or dynamics, so they belong there.

Also keep the **entry script** in `Main/` so it matches the repo style (`main_walk.m`, `main_hzd.m`, etc.).

# File Purpose in This Structure

### Entry script
`Main/main_HZD_optimization.m`

Runs:

- parameter loading
- fmincon optimization
- gait simulation
- plotting

---

### HZD math utilities
`Trajectory_Optimization/`

- `hzd_bezier.m`
- `hzd_phaseVariable.m`
- `hzd_h0_outputs.m`
- `hzd_jacobian_h0.m`
- `hzd_virtualConstraints.m`

These define the **HZD outputs and desired trajectories**.

---

### Simulation utilities

Also in `Trajectory_Optimization/`:

- `hzd_closedLoopDynamics.m`
- `hzd_footStrikeEvent.m`
- `hzd_simulateOneStep.m`
- `hzd_simulateNSteps.m`

They reuse existing repo functions:

- `D_matrix`
- `C_vector`
- `G_vector`
- `input_matrix`
- `rabbit_kinematics`
- `rabbit_impact_map`
- `rabbit_reset_map`

---

### Optimization components

Also in `Trajectory_Optimization/`:

- `hzd_unpackDecisionVars.m`
- `hzd_objectiveHZD.m`
- `hzd_constraintsHZD.m`

These implement the **fmincon problem**.

---

### Visualization

`Trajectory_Optimization/hzd_plotResults.m`

Plots:

- joint angles
- torques
- swing foot height
- phase variable

# MATLAB Path Requirement

Your `startup.m` probably already adds subfolders.  
If not, add:

```matlab
addpath(genpath(pwd));
```

so MATLAB sees:

- `Trajectory_Optimization`
- `Dynamics`
- `Model`
- `Contact`
